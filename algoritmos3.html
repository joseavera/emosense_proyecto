Bloque A: Fundamentos Lógicos y de Control

🧠 Pregunta 1: Control de Ciclo por Parada Temprana

📖 Contexto: Prevención del Sobreajuste (Overfitting)

En Machine Learning, si un modelo entrena demasiado tiempo, comienza a memorizar los datos en lugar de aprender patrones generales. Para evitar este "Sobreajuste", los ingenieros usan una técnica llamada Early Stopping (Parada Temprana), que interrumpe dinámicamente el entrenamiento justo cuando el modelo deja de mejorar, sin importar cuántos pasos faltaban.

Estás entrenando una Red Neuronal y notas que el error de validación sube drásticamente. Deseas implementar un algoritmo con un bucle que detenga el entrenamiento automáticamente si el error no mejora en 3 iteraciones continuas. ¿Qué tipo de control lógico requiere esta acción?

💡 Apunte de Repaso: Los ciclos estructurados como Mientras (While) evalúan una condición booleana antes de ejecutar su bloque. Son ideales para sistemas dinámicos donde no conocemos el número exacto de pasos previos.

🎯 Pregunta 2: Estructuras Anidadas en Visión Artificial

📖 Contexto: Árboles de Decisión y Umbrales Jerárquicos

Los sistemas de IA evalúan su entorno calculando un Nivel de Confianza (una probabilidad matemática de 0 a 1). Para que un coche autónomo tome una decisión segura, el algoritmo pasa esta probabilidad a través de múltiples filtros lógicos encadenados. Si la situación no es crítica, se evalúa el nivel de precaución medio, y así sucesivamente.

En el flujo de imágenes de un vehículo autónomo: Si el grado de confianza en la detección de un peatón es mayor a 0.85, ejecuta "Frenado de Emergencia". Si está entre 0.50 y 0.85, ejecuta "Reducir Velocidad". De lo contrario, "Mantener Marcha". ¿Cuál es el orden correcto de anidamiento estructural para resolverlo?

💡 Apunte de Repaso: Las estructuras condicionales anidadas permiten bifurcar el flujo de control en múltiples opciones exclusivas para clasificar el estado de las variables secuencialmente.

📊 Pregunta 3: Arreglos y Filtros de Datos (Data Preprocessing)

📖 Contexto: Preprocesamiento y Limpieza de Datos

La IA aprende de los datos, y aplica la regla de "basura que entra, basura que sale". Los datos sin procesar se almacenan en estructuras llamadas Arreglos (Vectores) y suelen contener errores o valores nulos (NaN). El primer trabajo de un algoritmo de IA es inspeccionar posición por posición para limpiar esta información.

Antes de alimentar un modelo, necesitas filtrar los valores nulos dentro de un arreglo unidimensional de 1000 registros. ¿Cuál es la estrategia algorítmica estándar?

💡 Apunte de Repaso: Los ciclos indexados (Para / For) son la herramienta óptima para recorrer arreglos porque conocemos con precisión matemática la longitud exacta de la lista de datos a procesar.

Bloque B: Laboratorio de Simulaciones Dinámicas

💻 Simulador 1: Umbral de Activación (Perceptrón)

📖 Contexto: La Neurona Artificial y Funciones de Activación

El Perceptrón es la unidad matemática fundamental de una Red Neuronal. Funciona sumando el "peso" de las señales que recibe. Si esa suma supera un límite específico llamado Umbral de Activación, la neurona "se enciende" (dispara un 1). Si no lo supera, se inhibe (permanece en 0).

Analiza el código. Modifica el valor en la casilla de entrada, presiona "Ejecutar Algoritmo" para observar el comportamiento y responde.

Consola PSeInt Virtual v4.0Entorno: Clasificación
Algoritmo ClasificadorPerceptron
    Definir entradaX, umbral Como Real;
    umbral <- 0.80;
    Leer entradaX;
    Si entradaX > umbral Entonces
        Escribir "ESTADO: Clase A (Activada)";
    Sino
        Escribir "ESTADO: Clase B (Inhibida)";
    FinSi
FinAlgoritmo

Pregunta 4: Tras correr el simulador ingresando exactamente el valor frontera 0.80, ¿cuál es el estado arrojado en la terminal?

💡 Apunte de Repaso: Un error muy común al diseñar IA es confundir los operadores relacionales. > busca exclusividad estricta, mientras que >= incluye el valor límite de la frontera.

🔁 Simulador 2: Descenso de Gradiente (Pasos Incrementales)

📖 Contexto: Tasa de Aprendizaje (Learning Rate)

Al optimizar un modelo mediante el algoritmo de Descenso de Gradiente, la IA da "saltos" iterativos para corregir su error matemático. El tamaño de cada salto se conoce como Tasa de Aprendizaje. Modificar el tamaño del incremento en un ciclo simula cómo el modelo se ajusta más rápido (o más lento) en cada iteración.

Prueba asignando diferentes valores límite y analiza la velocidad matemática con la que se actualiza el contador.

Consola PSeInt Virtual v4.0Entorno: Bucles Matriz
Algoritmo AjusteGradiente
    Definir iterador, limite Como Entero;
    iterador <- 1;
    Leer limite;
    Mientras iterador <= limite Hacer
        Escribir "Optimización en lote procesada.";
        iterador <- iterador + 3;
    FinMientras
FinAlgoritmo

Pregunta 5: Si corres la simulación asignando un límite de 7, ¿cuántas veces se imprime el mensaje de optimización?

💡 Apunte de Repaso: La variable de control (iterador <- iterador + 3) altera el flujo convencional de un ciclo permitiendo recorrer grandes porciones de memoria en mucho menor tiempo.

🛑 Simulador 3: Función de Pérdida (Validación de Signos)

📖 Contexto: Funciones de Costo y Error Absoluto

Para aprender, una Inteligencia Artificial mide constantemente qué tan equivocada está calculando una Función de Pérdida (Loss Function). Muchas veces lo que importa es el "volumen" del error y no si es positivo o negativo. Por ello, la lógica del algoritmo convierte los fallos negativos a positivos aplicando el equivalente matemático del Valor Absoluto.

Ingresa un número negativo o positivo para evaluar el cálculo final de la penalización del error.

Consola PSeInt Virtual v4.0Entorno: Funciones Costo
Algoritmo FiltroPerdida
    Definir score Como Real;
    Leer score;
    Si score < 0 Entonces
        score <- score * -1;
    FinSi
    Escribir "Magnitud absoluta del error: ", score;
FinAlgoritmo

Pregunta 6: Si el usuario ingresa un valor inicial de -15, ¿qué se mostrará finalmente impreso tras pasar por la estructura de selección?

💡 Apunte de Repaso: Multiplicar por -1 dentro de un condicional es la forma algorítmica fundamental de homogeneizar datos desviados antes de sumarlos al total del error del sistema.

🤖 Simulador 4: Contador Automático de Anomalías

📖 Contexto: Procesamiento por Lotes y Acumulación

En análisis de Big Data, procesar millones de datos simultáneamente sobrecarga la memoria. La solución de ingeniería es enviar pequeños bloques progresivos (batches). Para llevar la cuenta total de los resultados de cada bloque, la IA usa estructuras llamadas Variables Acumuladoras, que actualizan y guardan el peso sumado de cada iteración del ciclo.

Este simulador corre una estructura iterativa de tipo "Para" que acumula el valor en una simulación secuencial.

Consola PSeInt Virtual v4.0Entorno: Contador Ciclos
Algoritmo ContadorDeAnomalias
    Definir i, acumulador Como Entero;
    acumulador <- 0;
    Para i <- 1 Hasta 4 Con Paso 1 Hacer
        acumulador <- acumulador + i;
    FinPara
    Escribir "Anomalías acumuladas: ", acumulador;
FinAlgoritmo

Pregunta 7: Al finalizar todas las vueltas del ciclo 'Para', ¿cuál es el valor final almacenado en la variable 'acumulador'?

💡 Apunte de Repaso: Una variable acumuladora tiene como función sumar valores heterogéneos de manera progresiva sobre su propio valor guardado con anterioridad (acumulador <- acumulador + valor_nuevo).
